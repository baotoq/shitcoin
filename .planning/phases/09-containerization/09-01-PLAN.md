Create `.dockerignore` at project root with these exclusions:
- `.git` and git metadata (.gitignore, .gitmodules, .gitattributes)
- `data/` (runtime BoltDB state -- never bake into images)
- `wallets.json` (runtime wallet state)
- `node_modules` and `web/node_modules`

- Build artifacts (shitcoin binary, *.exe, *.test, *.out, coverage.html)
- Planning/docs (.planning, .agents, .claude, README.md, CLAUDE.md, *.md)
- Dockerfiles and docker-compose files (Dockerfile*, docker-compose*.yml)

Create `Dockerfile` at project root with two stages:

**Build stage** (golang:1.26-alpine):
- WORKDIR /build
- Copy go.mod and go.sum first, run `go mod download` (layer caching)
- Copy only needed source: cmd/, internal/
- Build with `CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o /app cmd/shitcoin/main.go`

**Runtime stage** (alpine:3.21):
- Create non-root user: `addgroup -g 1001 -S appgroup && adduser -S appuser -u 1001 -G appgroup`
- WORKDIR /app
- Copy binary from builder with --chown=appuser:appgroup
- Copy etc/shitcoin.yaml into /app/etc/shitcoin.yaml with --chown=appuser:appgroup (copy from build context, not builder stage)
- Create /app/data directory owned by appuser for runtime BoltDB storage
- USER appuser
- EXPOSE 8080
- CMD ["./app", "-f", "etc/shitcoin.yaml", "startnode"]

Key decisions per research:
- Use alpine:3.21 (not scratch) for shell/debugging access
- CGO_ENABLED=0 is mandatory (BoltDB pure Go build, avoids glibc dependency)
- -ldflags="-s -w" strips debug symbols (~30% smaller binary)
- Config file must be in image since binary expects `-f etc/shitcoin.yaml`
