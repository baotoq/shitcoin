<task type="auto">
  <name>Task 1: Create nginx.conf for SPA routing and reverse proxy</name>
  <files>web/nginx.conf</files>
  <action>
Create `web/nginx.conf` with:

1. **Map block** for WebSocket upgrade (must be outside server block):
   ```
   map $http_upgrade $connection_upgrade {
       default upgrade;
       ''      close;
   }
   ```

2. **Server block** listening on port 8080 (non-root compatible):
   - `server_name _;`
   - `root /usr/share/nginx/html;`
   - `index index.html;`

3. **SPA fallback** location `/`:
   - `try_files $uri $uri/ /index.html;`

4. **API reverse proxy** location `/api/`:
   - `proxy_pass http://backend:8080;`
   - Standard proxy headers: Host, X-Real-IP, X-Forwarded-For, X-Forwarded-Proto

5. **WebSocket reverse proxy** location `/ws`:
   - `proxy_pass http://backend:8080;`
   - `proxy_http_version 1.1;`
   - `proxy_set_header Upgrade $http_upgrade;`
   - `proxy_set_header Connection $connection_upgrade;`
   - Standard proxy headers: Host, X-Real-IP

The upstream hostname `backend` will resolve via K8s Service DNS (Phase 11). For standalone Docker testing, `--add-host backend:host-gateway` can be used.

<task type="auto">
  <name>Task 2: Create React frontend Dockerfile with nginx</name>
  <files>web/Dockerfile</files>
  <action>
Create `web/Dockerfile` with two stages:

**Build stage** (node:22-alpine):
- WORKDIR /app
- Copy package.json and package-lock.json first, run `npm ci` (layer caching)
- Copy all remaining source (COPY . .)
- Run `npm run build` to produce dist/ with Vite

**Runtime stage** (nginx:1.27-alpine):
- Create non-root user: `addgroup -g 1001 -S appgroup && adduser -S appuser -u 1001 -G appgroup`
- Fix nginx directory permissions for non-root: `chown -R appuser:appgroup /var/cache/nginx /var/log/nginx`
- Create and fix ownership of PID file: `touch /var/run/nginx.pid && chown appuser:appgroup /var/run/nginx.pid`
- Copy built assets: `COPY --from=builder /app/dist /usr/share/nginx/html`
- Copy nginx config: `COPY nginx.conf /etc/nginx/conf.d/default.conf`
- USER appuser
- EXPOSE 8080
- CMD ["nginx", "-g", "daemon off;"]

Build context is `web/` directory. The Dockerfile should be run with:
`docker build -t shitcoin-frontend web/`

This means all paths in COPY are relative to `web/`, so `COPY nginx.conf` works directly.
  </action>
