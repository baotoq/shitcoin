  <name>Task 1: Implement testnet CLI subcommand</name>
  <files>internal/handler/cli/testnet.go, internal/handler/cli/cli.go</files>
  <action>
    **Create `internal/handler/cli/testnet.go`:**

    1. Add `testnet` method to CLI struct in a new file `testnet.go`.
    2. FlagSet with:
       - `-nodes N` (default 3): number of nodes to spawn
       - `-base-port PORT` (default 3000): starting port (nodes use base, base+1, base+2...)
       - `-base-http-port PORT` (default 8080): starting HTTP port for REST/WS
    3. For each node i (0 to N-1):
       - Derive port: `basePort + i`
       - Derive HTTP port: `baseHttpPort + i`
       - Derive datadir: `data/node-{port}/`
       - Create wallet for node 0 (used for mining): directly use the wallet package to create a wallet and save it to the node's wallet file path. Import `wallet` and `jsonfile` packages to create wallet inline. Use `wallet.NewWallet()` and save via `jsonfile.NewWalletRepository(walletPath)`.

    4. Spawn each node as a child process using `exec.CommandContext`:
       - Binary: use `os.Args[0]` -- works for both compiled binary and `go run` result
       - Args: `-f {configPath} startnode -port {port} -datadir {datadir}`
       - Node 0 adds: `-mine {walletAddress}`
       - Nodes 1..N-1 add: `-peers localhost:{basePort}` (connect to node 0)
       - Set `cmd.SysProcAttr = &syscall.SysProcAttr{Setpgid: true}` for process group cleanup
       - Set up stdout/stderr pipes with `[node-{i}]` prefix using a goroutine that reads lines and prefixes them

    5. Print status table:
       ```
       Testnet started with N nodes:
         Node 0: port {port}, mining to {address} (seed node)
         Node 1: port {port+1}, connected to localhost:{port}
         ...
       Press Ctrl+C to stop all nodes.
       ```

    6. Set up signal handler (SIGINT, SIGTERM):
       - On signal, send SIGTERM to all child processes via process group kill
       - Wait for all processes to exit (with a 5-second timeout, then SIGKILL)
       - Do NOT clean up data directories automatically (user may want to inspect)

    7. Wait for all child processes. If any exits unexpectedly, log it.

    **Update `cli.go`:**
    8. Add `case "testnet"` to the `Run` switch statement, calling `c.testnet(args[1:])`.
    9. Add testnet to `printUsage()`: `"  testnet [-nodes N] [-base-port PORT] - Launch a local multi-node testnet"`

    **Important implementation notes:**
    - Use `exec.CommandContext(ctx, ...)` where ctx is cancelled on SIGINT/SIGTERM
    - Pipe stdout/stderr through a scanner goroutine that prefixes each line with `[node-{i}]`
    - Add `-config` flag to testnet (default "etc/shitcoin.yaml") for the config file path to pass to child processes
