    - TestDrainByFee: mempool with 3 txs at different fees drains in descending fee order
    - TestDrainByFeeMaxTxs: mempool with 5 txs, DrainByFee(2) returns only top-2 by fee; remaining 3 stay in pool
    - TestDrainByFeeZeroLimit: DrainByFee(0) returns all (backward compat)

  <action>
    **Config changes:**
    1. Add `HalvingInterval int \`json:",default=210000"\`` to `ConsensusConfig` in config.go
    2. Add `MaxBlockTxs int \`json:",default=100"\`` to `ConsensusConfig` in config.go
    3. Add corresponding fields to `ChainConfig` in chain.go: `HalvingInterval int`, `MaxBlockTxs int`
    4. Wire new fields in `service_context.go` where ChainConfig is built from ConsensusConfig
    5. Add `HalvingInterval: 10` to demo config `etc/shitcoin.yaml`

    **Halving (chain.go):**
    6. Add method `func (c *Chain) rewardAtHeight(height uint64) int64` using right-shift: `reward >> (height / halvingInterval)`. Guard halvings >= 64 returns 0. HalvingInterval <= 0 means no halving.
    7. Replace `c.config.BlockReward` with `c.rewardAtHeight(newHeight)` in MineBlock's coinbase creation
    8. Replace `c.config.BlockReward` with `c.rewardAtHeight(0)` in Initialize's genesis coinbase creation

    **Fees (tx/validator.go):**
    9. Add `fee int64` parameter to `CreateTransactionWithChange`. Change output becomes `inputSum - amount - fee`. Validation: `inputSum < amount + fee` returns ErrInsufficientFunds. If `change == 0`, skip change output. Update ALL callers (cli.go send, and any tests).

    **Fee-aware mempool (mempool.go):**
    10. Change internal storage from `map[block.Hash]*tx.Transaction` to `map[block.Hash]*mempoolEntry` where `mempoolEntry` is `struct { tx *tx.Transaction; fee int64 }`.
    11. Modify `Add()` to accept `fee int64` as second parameter: `func (m *Mempool) Add(transaction *tx.Transaction, fee int64) error`. Store fee in entry. Update `MempoolAdder` interface in chain.go accordingly (add fee param or use 0 for reorg re-adds).

    Actually, to avoid breaking MempoolAdder interface used in Reorganize, keep `Add(transaction *tx.Transaction) error` and add `AddWithFee(transaction *tx.Transaction, fee int64) error`. The `Add` method calls `AddWithFee` with fee=0. CLI send computes fee and calls `AddWithFee`.

    12. Add `func (m *Mempool) AddWithFee(transaction *tx.Transaction, fee int64) error` that stores the fee.
    13. Make `Add()` delegate to `AddWithFee(transaction, 0)`.
    14. Add `func (m *Mempool) DrainByFee(maxTxs int) []*tx.Transaction`: sort entries by fee descending using `slices.SortFunc`, take top maxTxs (0 = all), return transactions, clean up maps. Return total fees as second return value: `func (m *Mempool) DrainByFee(maxTxs int) ([]*tx.Transaction, int64)`.
    15. Update `DrainAll()` to call `DrainByFee(0)` internally and discard the fee total (backward compat).
    16. Update all other Mempool methods (GetByID, Transactions, Count, Remove) to work with mempoolEntry map.

    **MineBlock fee collection (chain.go):**
    17. Change MineBlock signature to accept a totalFees parameter: No -- better approach: MineBlock receives txs and totalFees. Actually simplest: accept `[]*tx.Transaction` and `totalFees int64`. Update: `func (c *Chain) MineBlock(ctx context.Context, minerAddress string, txs []*tx.Transaction, totalFees int64) (*block.Block, error)`. Coinbase reward = `c.rewardAtHeight(newHeight) + totalFees`.
    18. Update all MineBlock callers (cli.go mine, signal.go autoMine, signal.go autoMineWithP2P, P2P relay handleBlock) to pass totalFees. For auto-mine and manual mine, use DrainByFee to get both txs and totalFees. For P2P received blocks, pass 0 (fees already in coinbase from sender).

    **CLI send -fee flag:**
    19. Add `-fee` flag to send command's FlagSet (default 0). Pass fee to CreateTransactionWithChange. Compute fee for mempool: `fee` value is directly the fee amount. Call `AddWithFee(transaction, *fee)`.

    **Validation test adjustments:**
    20. Update ValidateCoinbase callers if needed (currently only used in tests).
