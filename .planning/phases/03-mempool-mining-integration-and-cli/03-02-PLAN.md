    **1. Add `send` command to CLI** (CLI-04):
    - Parse flags: `-from`, `-to`, `-amount` (int64, in satoshis) using `flag.NewFlagSet("send", flag.ExitOnError)`
    - Initialize chain: `cli.svc.Chain.Initialize(ctx, "")`
    - Load sender wallet: `cli.svc.WalletRepo.GetByAddress(from)`
    - Get sender's UTXOs: `cli.svc.UTXOSet.GetByAddress(from)`
    - Select UTXOs to cover amount (simple greedy: iterate UTXOs, accumulate until total >= amount; if insufficient, print error and exit)
    - Build TxInputs from selected UTXOs: `tx.NewTxInput(utxo.TxID(), utxo.Vout())`
    - Collect input values ([]int64) from selected UTXOs
    - Create transaction: `tx.CreateTransactionWithChange(inputs, inputValues, to, amount, from)` -- change goes back to sender
    - Sign: `tx.SignTransaction(transaction, senderWallet.PrivateKey())`
    - Add to mempool: `cli.svc.Mempool.Add(transaction)`
    - Print: `Transaction {txID[:16]}... added to mempool`

    **2. Add `mine` command to CLI** (CLI-05, MINE-04):
    - Parse `-address` flag (miner address to receive block reward)
    - Initialize chain: `cli.svc.Chain.Initialize(ctx, address)` -- uses address for genesis coinbase if chain is new
    - Drain mempool: `txs := cli.svc.Mempool.DrainAll()`
    - Mine: `blk, err := cli.svc.Chain.MineBlock(ctx, address, txs)`
    - Print: `Mined block #{height} with {len(txs)} transaction(s)` and block hash
    - After mining, print updated chain height

    **3. Add `startnode` command to CLI** (CLI-07, MINE-05):
    - Parse flags: `-port` (int, stored for future P2P use), `-mine` (string, miner address -- if set, enables auto-mining)
    - Initialize chain: `cli.svc.Chain.Initialize(ctx, mineAddr)` if mine address provided, otherwise `Initialize(ctx, "")`
    - Print: `Node started. Chain height: {height}`
    - If `-mine` address provided:
      - Print: `Auto-mining enabled for address: {address}`
      - Set up context cancellation with `context.WithCancel`
      - Set up signal handler: `signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM)` in goroutine that calls `cancel()` on signal
      - Run auto-mine loop:
        ```go
        for {
            select {
            case <-ctx.Done():
                fmt.Println("Mining stopped.")
                return
            default:
                txs := cli.svc.Mempool.DrainAll()
                blk, err := cli.svc.Chain.MineBlock(ctx, mineAddr, txs)
                if err != nil {
                    if ctx.Err() != nil { return }
                    fmt.Printf("Mining error: %v\n", err)
                    continue
                }
                fmt.Printf("Mined block #%d (%s) with %d tx\n", blk.Height(), blk.Hash().String()[:16], len(txs))
            }
        }
        ```
    - If no `-mine` address: print `No mining address provided. Node running idle. Press Ctrl+C to stop.` and block on signal channel

    **4. Replace `cmd/shitcoin/main.go`:**
    - Remove the entire demo mining loop
    - New main.go:
      ```go
      func main() {
          configFile := flag.String("f", "etc/shitcoin.yaml", "the config file")
          flag.Parse()

          logx.Disable()
          stat.DisableLog()

          var c config.Config
          conf.MustLoad(*configFile, &c)
          c.Consensus.ApplyDefaults()

          serviceCtx := svc.NewServiceContext(c)
          defer serviceCtx.Close()

          app := cli.New(serviceCtx)
          app.Run()
      }
      ```
    - The CLI's Run() method handles os.Args[1] dispatch, so main.go just wires config -> ServiceContext -> CLI

    NOTE on os.Args: The `-f` config flag is parsed by `flag.Parse()` in main. CLI subcommands use `os.Args` starting from index 1. Since `flag.Parse()` consumes `-f`, the subcommand will be in `flag.Args()`. Alternatively, check if first non-flag arg is the subcommand. To keep it simple: use `os.Args` directly -- the user runs `./shitcoin createwallet` (no -f flag normally) or `./shitcoin -f custom.yaml createwallet`. For the `-f` case, use `flag.Args()` to get remaining args after flag parsing. Pass `flag.Args()` to `cli.Run()` instead of reading `os.Args` directly:
    ```go
    app.Run(flag.Args())
    ```
    And in CLI:
    ```go
    func (c *CLI) Run(args []string) {
        if len(args) < 1 { c.printUsage(); os.Exit(1) }
        switch args[0] { ... }
    }
    ```
    Each subcommand that needs its own flags creates a `flag.NewFlagSet` and parses `args[1:]`.
