    - TestAdd_DoubleSpend: Add two transactions spending same UTXO -> second returns ErrDoubleSpend
    - TestAdd_InvalidSignature: Add unsigned transaction -> returns ErrInvalidSignature
    - TestAdd_UTXONotFound: Add transaction referencing nonexistent UTXO -> returns ErrUTXONotFound

    - TestRemove: Add 2 transactions, Remove one by ID -> Count() == 1, removed one gone
    - TestTransactions: Add 2 transactions -> Transactions() returns both (read-only view)
    - TestConcurrentAccess: 10 goroutines adding transactions concurrently -> no race condition (with -race flag)

    Merkle root tests (internal/domain/block/merkle_test.go):
    - TestMerkleRoot_Empty: No hashes -> returns Hash{} (zero hash)
    - TestMerkleRoot_Single: One hash -> returns DoubleSHA256 of that hash concatenated with itself (Bitcoin convention: single leaf is hashed with itself)
    - TestMerkleRoot_Two: Two hashes -> returns DoubleSHA256(hash1 + hash2)
    - TestMerkleRoot_Odd: Three hashes -> duplicates last, computes as four
    - TestMerkleRoot_Even: Four hashes -> standard binary tree
    - TestMerkleRoot_Deterministic: Same inputs always produce same output
  </behavior>
  <action>
    **Mempool package** (`internal/domain/mempool/`):

    1. Create `errors.go` with sentinel errors: `ErrDuplicate`, `ErrDoubleSpend`, `ErrInvalidSignature`, `ErrUTXONotFound`.

    2. Create `mempool.go` with:
       - `Mempool` struct: `sync.RWMutex`, `txs map[block.Hash]*tx.Transaction`, `utxoSet *utxo.Set`
       - `New(utxoSet *utxo.Set) *Mempool` constructor
       - `Add(transaction *tx.Transaction) error`: acquires write lock, checks duplicate (by tx ID), checks double-spend against pool (compare input txID:vout pairs), verifies signature via `tx.VerifyTransaction()`, verifies UTXO existence via `utxoSet.Get()`, adds to map
       - `DrainAll() []*tx.Transaction`: acquires write lock, copies all values to slice, resets map, returns slice
       - `Remove(txIDs []block.Hash)`: acquires write lock, deletes by IDs
       - `Count() int`: acquires read lock, returns len(txs)
       - `Transactions() []*tx.Transaction`: acquires read lock, returns copy of all values (read-only view for display)

    3. Create `mempool_test.go` with the behaviors listed above. For tests, create a helper that builds a valid signed transaction against an in-memory UTXO set:
       - Use `btcec.NewPrivateKey()` to create a key
       - Create a coinbase TX, apply it to UTXO set (via set.ApplyBlock)
       - Build a spending TX referencing the coinbase output, sign with the private key
       - Use `utxo.NewSet()` with the in-memory UTXO repository from `utxo/set_test.go` pattern (or create a simple in-memory repo)

    NOTE: The mempool needs a `*utxo.Set` for validation. For tests, create a minimal in-memory UTXO repository implementing `utxo.Repository`. Look at existing `internal/domain/utxo/set_test.go` for the pattern.

    **Merkle root** (`internal/domain/block/merkle.go`):

    4. Create `merkle.go` with `ComputeMerkleRoot(txHashes []Hash) Hash`:
       - Empty slice -> return `Hash{}`
       - Copy input to avoid mutation
       - Loop while len > 1: if odd count, duplicate last hash; pair-wise combine via `DoubleSHA256(hash[i].Bytes() + hash[i+1].Bytes())`, collect into next level
       - Return final hash

    5. Create `merkle_test.go` with the behaviors listed above.
