2. Migrate each of the 7 test files following these rules (from CONTEXT.md locked decisions):
   - Replace `if got != want { t.Errorf(...) }` with `assert.Equal(t, want, got)` -- note: expected FIRST, actual SECOND (testify convention).
   - Replace `if err != nil { t.Fatalf(...) }` with `require.NoError(t, err)` -- use `require` for setup/preconditions.
   - Replace `if err == nil { t.Fatal(...) }` with `require.Error(t, err)`.
   - Replace `if result == nil { t.Fatal(...) }` with `require.NotNil(t, result)`.
   - Replace `if !valid { t.Error(...) }` with `assert.True(t, valid)`.
   - Replace `if len(x) != N { ... }` with `require.Len(t, x, N)` when used as precondition, `assert.Len(t, x, N)` otherwise.
   - Replace zero-value checks like `if root != (Hash{})` with `assert.Equal(t, Hash{}, root)`.
   - Use `assert.ErrorIs(t, err, specificErr)` for error type comparisons.
   - For table-driven tests with `t.Run`, add `assert := assert.New(t)` and `require := require.New(t)` at the top of each subtest, then use `assert.Equal(want, got)` (no `t` parameter).
   - Only add assertion messages when the assertion is not self-explanatory. Do NOT add messages to every assertion.
   - Add imports: `"github.com/stretchr/testify/assert"` and/or `"github.com/stretchr/testify/require"` as needed.
   - Remove any custom assertion helper functions that testify now replaces.
