  <name>Task 1: Create Dockerfile.dev, kind cluster config, and update .gitignore</name>
  <files>Dockerfile.dev, deploy/k8s/kind-cluster.yaml, .gitignore</files>
  <action>
1. Create `Dockerfile.dev` in project root -- a lightweight Dockerfile for Tilt's binary sync pattern. This skips the Go build stage entirely and just copies a pre-compiled binary:

```dockerfile
FROM alpine:3.21
RUN addgroup -g 1001 -S appgroup && adduser -S appuser -u 1001 -G appgroup
WORKDIR /app
COPY build/shitcoin /app/shitcoin
COPY etc/shitcoin.yaml /app/etc/shitcoin.yaml
USER appuser
ENTRYPOINT ["/app/shitcoin"]
```

Why a separate Dockerfile: The production Dockerfile copies `cmd/` and `internal/` for in-image compilation. Tilt's `only=['./build', './etc']` would break those COPY steps. A dev Dockerfile that takes just the pre-compiled binary is the standard pattern.

2. Create `deploy/k8s/kind-cluster.yaml` -- minimal kind cluster config:

```yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
name: shitcoin
nodes:
  - role: control-plane
```

No `extraPortMappings` needed -- Tilt handles port forwarding.

3. Update `.gitignore` -- add `build/` directory entry for compiled binary output:

Append `build/` to the project section of .gitignore (near the existing `bin/` entry).

<task type="auto">
  <name>Task 2: Create Tiltfile with live-update for backend and frontend</name>
  <files>Tiltfile</files>
  <action>
Create `Tiltfile` in project root using the research-recommended patterns:

**Backend (Go) -- compile locally, sync binary pattern:**
- Load `restart_process` extension: `load('ext://restart_process', 'docker_build_with_restart')`
- `local_resource('backend-compile', ...)` compiles Go binary to `build/shitcoin` with `CGO_ENABLED=0 GOOS=linux` (omit GOARCH to default to host arch, avoiding QEMU issues on Apple Silicon)
- `deps=['cmd/', 'internal/', 'go.mod', 'go.sum']` to watch Go source files
- `docker_build_with_restart('shitcoin-backend', '.', ...)` with:
  - `entrypoint=['/app/shitcoin', '-f', '/app/etc/shitcoin.yaml', 'startnode']`
  - `dockerfile='Dockerfile.dev'`
  - `only=['./build', './etc']`
  - `live_update=[sync('./build/shitcoin', '/app/shitcoin')]`

**Frontend (React) -- rebuild and sync assets pattern:**
- `local_resource('frontend-build', 'cd web && npm run build', deps=['web/src/', 'web/index.html', 'web/vite.config.ts', 'web/tailwind.config.ts'])` to build locally
- `docker_build('shitcoin-frontend', 'web', ...)` with:
  - `dockerfile='web/Dockerfile'`
  - `only=['./dist', './nginx.conf']`
  - `live_update=[fall_back_on(['web/package.json', 'web/package-lock.json']), sync('./web/dist', '/usr/share/nginx/html')]`

**Kubernetes integration:**
- `k8s_yaml(kustomize('deploy/k8s/overlays/dev'))` to load existing dev overlay
- `k8s_resource('backend', port_forwards=[port_forward(8080, 8080, name='Backend API')], resource_deps=['backend-compile'])`
- `k8s_resource('frontend', port_forwards=[port_forward(3000, 8080, name='Frontend')], resource_deps=['frontend-build'])`

Add clear comments explaining each section. The Tiltfile should be readable as documentation of the dev workflow.
