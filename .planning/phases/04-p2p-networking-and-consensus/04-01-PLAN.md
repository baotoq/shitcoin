  <behavior>
    - WriteMessage followed by ReadMessage round-trips a Message correctly
    - ReadMessage returns ErrMessageTooLarge when length exceeds MaxMessageSize (10MB)
    - ReadMessage returns error on truncated/malformed frames
    - VersionPayload JSON marshals/unmarshals correctly with all fields (Version, Height, GenesisHash, ListenPort)
    - Peer writeLoop sends messages from sendCh to connection, stops on channel close
    - Peer readLoop reads messages from connection and calls handler callback
  </behavior>
  <action>
    Create the `internal/domain/p2p/` package with these files:

    **errors.go**: Sentinel errors:
    - `ErrMessageTooLarge` (message exceeds MaxMessageSize)
    - `ErrHandshakeFailed` (version handshake failed)
    - `ErrIncompatibleGenesis` (genesis hash mismatch)
    - `ErrProtocolViolation` (unexpected message in current state)

    **message.go**: Message type and command constants:
    - `const MaxMessageSize = 10 * 1024 * 1024` (10MB)
    - `const ProtocolVersion uint32 = 1`
    - Command byte constants: `CmdVersion=0x01`, `CmdVerack=0x02`, `CmdGetBlocks=0x03`, `CmdInv=0x04`, `CmdGetData=0x05`, `CmdBlock=0x06`, `CmdTx=0x07`
    - `type Message struct { Command byte; Payload []byte }`
    - `func NewMessage(cmd byte, payload any) (Message, error)` -- JSON-marshals payload into Message
    - Payload structs: `VersionPayload{Version uint32, Height uint64, GenesisHash string, ListenPort int}`, `InvPayload{Type string, Hashes []string}`, `GetBlocksPayload{StartHeight uint64, EndHeight uint64}`

    **protocol.go**: Length-prefixed TCP framing:
    - Wire format: `[4-byte big-endian length][1-byte command][JSON payload]`
    - `func WriteMessage(w io.Writer, msg Message) error` -- writes length-prefixed frame
    - `func ReadMessage(r io.Reader) (Message, error)` -- reads frame, validates size, returns Message
    - Use `encoding/binary.BigEndian` and `io.ReadFull`

    **peer.go**: Peer connection management:
    - `type Peer struct` with fields: `conn net.Conn`, `addr string`, `sendCh chan Message` (buffered, cap 64), `height uint64`, `version uint32`, `done chan struct{}`
    - `func NewPeer(conn net.Conn, addr string) *Peer`
    - `func (p *Peer) Start(handler func(*Peer, Message))` -- launches read and write goroutines
    - `func (p *Peer) Send(msg Message)` -- non-blocking send via select with default (drop if full)
    - `func (p *Peer) Stop()` -- closes connection and done channel
    - Write goroutine: range over sendCh, call WriteMessage, return on error
    - Read goroutine: loop ReadMessage, call handler, return on error (which triggers Stop)

    **p2p_test.go**: Tests using `net.Pipe()` for in-memory connections:
    - Test WriteMessage/ReadMessage round-trip
    - Test ErrMessageTooLarge on oversized length header
    - Test VersionPayload serialization
    - Test Peer Start/Send/Stop lifecycle

  <behavior>
    - Server.Start listens on configured TCP port and accepts connections
    - Server.Connect dials a peer address and initiates outbound handshake
    - Version handshake: initiator sends Version, responder sends Version+Verack, initiator sends Verack
    - After handshake, peer is added to peer registry with height from VersionPayload
    - Peers with different GenesisHash are disconnected during handshake
    - Server.Broadcast sends a message to all peers except excluded address
    - Server.Stop gracefully shuts down listener and all peer connections
    - startnode CLI with -port flag derives unique data directory (data/node-{port}/)
    - startnode CLI with -peers flag connects to comma-separated peer addresses
  </behavior>
  <action>
    **server.go**: TCP server and peer manager:
    - `type Server struct` with fields: `mu sync.RWMutex`, `peers map[string]*Peer`, `listener net.Listener`, `chain *chain.Chain`, `mempool *mempool.Mempool`, `utxoSet *utxo.Set`, `chainRepo chain.Repository`, `listenPort int`, `ctx context.Context`, `cancel context.CancelFunc`
    - `func NewServer(chain *chain.Chain, mempool *mempool.Mempool, utxoSet *utxo.Set, chainRepo chain.Repository, port int) *Server`
    - `func (s *Server) Start(ctx context.Context) error` -- creates TCP listener on `localhost:{port}`, launches accept loop goroutine
    - Accept loop: `for { conn, err := listener.Accept(); go s.handleInbound(conn) }` -- use ctx for shutdown
    - `func (s *Server) Connect(addr string) error` -- dials TCP, creates Peer, performs outbound handshake
    - `func (s *Server) handleInbound(conn net.Conn)` -- creates Peer, waits for inbound Version message, responds
    - `func (s *Server) Broadcast(msg Message, excludeAddr string)` -- iterates peers, sends to all except excluded
    - `func (s *Server) Stop()` -- cancels context, closes listener, stops all peers
    - `func (s *Server) PeerCount() int` -- returns number of connected peers
    - Handshake flow:
      - Outbound: send Version(our height, genesis hash, protocol version, listen port) -> wait for Version -> check genesis match -> send Verack -> wait for Verack -> register peer
      - Inbound: wait for Version -> check genesis match -> send Version(our info) + Verack -> wait for Verack -> register peer
    - Set 10-second deadline on handshake reads/writes via `conn.SetDeadline`; clear deadline after handshake

    **handler.go**: Message dispatch:
    - `func (s *Server) handleMessage(peer *Peer, msg Message)` -- switch on msg.Command, dispatch to handler methods
    - For now, only handle CmdVersion and CmdVerack in the handler (relay commands added in Plan 02)
    - Log unknown commands with `log/slog`

    - Create a NEW ServiceContext with the overridden config (so each node gets its own DB)
