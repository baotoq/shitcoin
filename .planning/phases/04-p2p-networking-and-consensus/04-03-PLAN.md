<objective>
Implement initial block download (IBD) so new nodes synchronize the full chain from existing peers.

Purpose: Without IBD, every node must mine from genesis. This enables nodes to join an existing network and catch up to the current chain state.
Output: A new node connects to a peer, detects it has a longer chain, downloads all blocks sequentially, validates and applies each one, and reaches the same chain height.
</objective>

<task type="auto" tdd="true">
  <name>Task 2: Initial block download sync logic triggered after handshake</name>
  <files>
    internal/domain/p2p/sync.go
    internal/domain/p2p/server.go
    internal/domain/p2p/p2p_test.go

  <behavior>
    - After handshake, if peer's height > local height, IBD is triggered automatically
    - During IBD, server.syncing flag is true (prevents mining)
    - IBD sends GetBlocks for batches of blocks, applies each received block sequentially
    - After IBD completes, local chain height matches peer's height
    - UTXO set is correctly populated after IBD (balances queryable)
    - If IBD receives an invalid block, it aborts sync and disconnects the peer
  </behavior>
  <action>
    **sync.go**: Create IBD sync logic:
    - `func (s *Server) startSync(peer *Peer)` -- called after handshake when peer.height > local height
    - Sets `s.syncing = true` (atomic bool) to signal mining should be paused
    - Sends `CmdGetBlocks{StartHeight: localHeight + 1, EndHeight: 0}` to peer
    - Blocks are received asynchronously via the CmdBlock handler
    - The sync coordinator tracks expected height range and counts received blocks
    - For each received block during sync:
      1. Validate PoW with `pow.Validate()`
      2. Verify prevHash matches current chain tip
      3. Extract transactions from block payload
      4. Apply UTXO changes via `utxoSet.ApplyBlock()`
      5. Save via `chainRepo.SaveBlockWithUTXOs()`
      6. Update chain tip
    - When all blocks in the batch are received, check if peer has more (peer.height > new local height), send another GetBlocks if needed
    - When fully caught up, set `s.syncing = false`
    - If any block is invalid, abort sync, disconnect peer, set syncing = false

    **server.go changes**:
    - Add `syncing atomic.Bool` field
    - Add `func (s *Server) IsSyncing() bool` -- returns syncing state
    - Modify handshake completion: after registering peer, if peer.height > local chain height, call `go s.startSync(peer)`
    - The CmdBlock handler (from Plan 02) needs to differentiate between "block from sync" vs "live block relay". Use a `syncState` struct on the server that tracks: syncing bool, expected height, target height, source peer addr. When syncing and block arrives from sync source peer at expected height, route to sync processing. Otherwise handle as live relay.

    **p2p_test.go additions**:
    - `TestInitialBlockDownload`: Node A has 5 blocks. Node B starts fresh. B connects to A. After sync, B has 5 blocks and same tip hash as A.
    - `TestIBDUTXOConsistency`: After IBD, query balance on synced node matches the original node's balance.
    - `TestIBDInvalidBlock`: Serve a block with bad PoW during sync. Verify sync aborts and peer is disconnected.
    - Test helper: `mineTestBlocks(t, chain, n, minerAddr)` to create a chain of N blocks for testing.
  </action>
