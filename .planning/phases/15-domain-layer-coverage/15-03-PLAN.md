**Create handler_test.go** (new file, package p2p_test):

1. **TestHandleTx_ValidTransaction** -- Send a valid transaction via CmdTx to a server:
   - Set up a test node with chain, mempool, UTXO set
   - Create a signed transaction using testutil builders
   - Send it via raw connection using WriteMessage
   - Use `require.Eventually` to assert mempool.Count() increases

2. **TestHandleTx_InvalidPayload** -- Send malformed JSON as CmdTx payload:
   - Send a Message with CmdTx command but invalid JSON payload
   - Verify mempool count remains 0 (message rejected, no crash)

3. **TestHandleTx_RejectedByMempool** -- Send a transaction that fails mempool validation (e.g., double-spend):
   - Add a transaction to mempool, then send another spending the same UTXO
   - Verify mempool count does not increase

4. **TestHandleGetData_Block** -- Send CmdGetData requesting a block:
   - Initialize chain with genesis block
   - Send getdata message requesting the genesis block hash with type "block"
   - Read response, verify a CmdBlock message is received with correct block data

5. **TestHandleGetData_Tx** -- Send CmdGetData requesting a transaction from mempool:
   - Add a transaction to mempool
   - Send getdata requesting that tx hash with type "tx"
   - Read response, verify CmdTx message received

6. **TestHandleGetData_NotFound** -- Send CmdGetData for a hash that doesn't exist:
   - Send getdata with a random hash
   - Verify no crash, connection stays alive
