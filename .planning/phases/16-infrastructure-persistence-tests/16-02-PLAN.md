Add the following tests to wallet_repo_test.go:

**TestWalletRepo_CorruptFile:** Create a temp dir, write invalid JSON `{not valid json` to wallets.json, call NewWalletRepo with that path. Assert error is returned (require.Error).

**TestWalletRepo_InvalidPrivateKey:** Create a temp dir, write valid JSON with an invalid private_key_hex value: `{"wallets":[{"address":"1test","private_key_hex":"not-a-hex-key"}]}`. Call NewWalletRepo. Assert error is returned (the ReconstructWallet call will fail).

**TestWalletRepo_UnreadableFile:** Create a temp dir, write a valid file, then chmod it to 0000 (write-only/no-read). Call NewWalletRepo. Assert error is returned. Use t.Cleanup to restore permissions (os.Chmod back to 0644) so t.TempDir cleanup succeeds.

**TestWalletRepo_FlushError_ReadOnlyDir:** Create a temp dir, create a valid WalletRepo pointing to a file inside a subdirectory. Create a wallet and save it successfully first (to confirm it works). Then chmod the subdirectory to 0555 (read-only). Create a new wallet and call Save -- assert error is returned because flush cannot write the temp file. Use t.Cleanup to restore directory permissions (os.Chmod back to 0755) so t.TempDir cleanup succeeds.

All tests use t.TempDir() for isolation. Permission-based tests should use `runtime.GOOS` check or be skipped on Windows (though darwin/linux is fine per the project).
