WebSocket test pattern from research:
```go
server := httptest.NewServer(ServeWs(hub))
defer server.Close()
wsURL := "ws" + strings.TrimPrefix(server.URL, "http")
conn, _, err := websocket.DefaultDialer.Dial(wsURL, nil)
require.NoError(t, err)
defer conn.Close()
```

Note: Hub.Run() starts an infinite goroutine with no Stop(). For tests this is acceptable -- the goroutine leaks but tests finish quickly. If -race warns, the server.Close() and conn.Close() handle cleanup sufficiently.

  <action>
Create `handler_test.go` (package ws) with integration tests using httptest.Server + gorilla/websocket.DefaultDialer:

- TestServeWs_ClientReceivesBroadcast:
  1. Create events.NewBus() and NewHub(bus).
  2. Start httptest.NewServer(ServeWs(hub)), defer server.Close().
  3. Convert URL: wsURL := "ws" + strings.TrimPrefix(server.URL, "http").
  4. Dial with websocket.DefaultDialer.Dial(wsURL, nil), defer conn.Close().
  5. Use require.Eventually to wait for client to be registered in hub (check hub.mu.RLock/clients length).
  6. Publish event: bus.Publish(events.Event{Type: events.EventNewBlock, Payload: map[string]string{"hash": "abc123"}}).
  7. Set conn.SetReadDeadline(time.Now().Add(2*time.Second)).
  8. Read message: _, msg, err := conn.ReadMessage(). Assert no error.
  9. Unmarshal into WSMessage, assert Type == "new_block".

- TestServeWs_ClientDisconnectUnregisters:
  1. Same setup as above.
  2. Wait for client registration with require.Eventually.
  3. Close the connection: conn.Close().
  4. Use require.Eventually to verify hub.clients is empty (client was unregistered).

- TestServeWs_MultipleClients:
  1. Create hub + server.
  2. Dial two WebSocket connections.
  3. Wait for both to register (require.Eventually, len(hub.clients) == 2).
  4. Broadcast a message via hub.broadcast <- []byte(`{"type":"test"}`).
  5. Read from both connections with deadline, assert both receive the message.

- TestServeWs_EventBusIntegration:
  1. Create bus + hub + server.
  2. Dial one connection.
  3. Publish multiple events on bus (e.g., EventNewBlock, then EventMempoolChanged).
  4. Read messages from connection, verify both arrive with correct types.

For all require.Eventually calls, use 1-second timeout with 10ms poll interval. This replaces time.Sleep per STATE.md decision.
