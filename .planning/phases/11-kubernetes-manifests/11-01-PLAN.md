  <action>
Create directory deploy/k8s/base/ and write these 5 manifests:

1. **backend-deployment.yaml**: apps/v1 Deployment named "backend" with labels app=shitcoin, component=backend.
   - replicas: 1
   - strategy: type: Recreate (BoltDB single-writer safety)
   - Container named "backend", image: shitcoin-backend, containerPort: 8080
   - volumeMounts: "data" at /app/data, "config" at /app/etc (readOnly: true)
   - livenessProbe: httpGet /api/status port 8080, initialDelaySeconds: 10, periodSeconds: 30, failureThreshold: 3
   - readinessProbe: httpGet /api/status port 8080, initialDelaySeconds: 5, periodSeconds: 10, failureThreshold: 2
   - resources: requests cpu=100m memory=128Mi, limits cpu=500m memory=256Mi
   - volumes: "data" from PVC "backend-data", "config" from configMap "shitcoin-config"

2. **backend-service.yaml**: v1 Service named "backend" (MUST be "backend" -- nginx.conf proxies to http://backend:8080). ClusterIP, selector app=shitcoin component=backend, port 8080 targetPort 8080.

3. **backend-pvc.yaml**: v1 PVC named "backend-data", labels app=shitcoin component=backend, accessModes: [ReadWriteOnce], storage: 1Gi. Do NOT specify storageClassName (let cluster default handle it).

4. **frontend-deployment.yaml**: apps/v1 Deployment named "frontend", labels app=shitcoin component=frontend. replicas: 1, container named "frontend", image: shitcoin-frontend, containerPort: 8080. No volumes needed. Resources: requests cpu=50m memory=64Mi, limits cpu=200m memory=128Mi.

5. **frontend-service.yaml**: v1 Service named "frontend", ClusterIP, selector app=shitcoin component=frontend, port 8080 targetPort 8080.

All manifests use consistent labels: app=shitcoin, component={backend|frontend}.
