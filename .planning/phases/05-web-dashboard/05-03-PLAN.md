  <behavior>

    - Test 2: Hub unregisters client, removes from map, and closes send channel
    - Test 3: Hub broadcasts message to all registered clients
    - Test 4: Hub drops message when client send channel is full (non-blocking)
    - Test 5: Hub forwards events from event bus subscription to all clients as JSON
  </behavior>
  <action>
    First: `go get github.com/gorilla/websocket@v1.5.3`

    Create `internal/handler/ws/hub.go`:
    - `Hub` struct with `clients map[*Client]bool`, `broadcast chan []byte`, `register chan *Client`, `unregister chan *Client`, `eventBus *events.Bus`
    - `NewHub(bus *events.Bus) *Hub`
    - `Run()` method -- goroutine with select loop: register, unregister, broadcast (following gorilla chat example pattern from research)
    - On unregister: delete client, close client.send
    - On broadcast: iterate clients, non-blocking send, drop slow clients (close and delete)
    - Start a second goroutine that subscribes to `eventBus`, converts each `Event` to `WSMessage` JSON bytes, and sends to `broadcast` channel
    - `ServeHTTP(w, r)` method on Hub that delegates to `handler.go`

    Create `internal/handler/ws/client.go`:
    - `Client` struct with `hub *Hub`, `conn *websocket.Conn`, `send chan []byte` (buffered, cap 256)
    - `readPump()` -- goroutine reading from WebSocket (just consume and discard client messages, or handle ping/pong). On read error, unregister from hub.
    - `writePump()` -- goroutine writing from send channel to WebSocket. Handle write errors by closing. Set write deadline. Support ping/pong with 60s pong wait, 54s ping period.
    - Constants: `writeWait = 10s`, `pongWait = 60s`, `pingPeriod = 54s`, `maxMessageSize = 512`

    Create `internal/handler/ws/handler.go`:
    - `var upgrader = websocket.Upgrader{CheckOrigin: func(r) bool { return true }}` -- allow all origins for dev
    - `ServeWs(hub *Hub) http.HandlerFunc` -- upgrade HTTP to WebSocket, create Client, register with hub, start readPump and writePump goroutines

    Create `internal/handler/ws/hub_test.go`:
    - Test hub register/unregister/broadcast using channels directly (no WebSocket needed for unit tests)
    - Test event bus integration: publish event to bus, verify it arrives as JSON bytes on broadcast channel
  </action>
