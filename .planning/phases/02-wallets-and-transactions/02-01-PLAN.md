Create the wallet domain package with ECDSA key pair management using btcec/v2 (secp256k1) and Bitcoin-style P2PKH address derivation with hand-rolled Base58Check encoding, plus JSON file persistence.

Purpose: Wallets are the identity layer -- every transaction requires a sender address (for signing) and recipient address (for outputs). This plan is foundational for TX-03 through TX-06.
Output: Wallet entity, address derivation, Base58Check codec, wallet repository with JSON file implementation, and comprehensive tests.
</objective>

    - Base58Encode: input with leading zero bytes produces leading '1' characters
    - Base58Decode: round-trips with Base58Encode for various inputs
    - Base58CheckEncode: version 0x00 + known payload produces expected address
    - Base58CheckDecode: valid address decodes to correct version and payload
    - Base58CheckDecode: invalid checksum returns error
    - PubKeyToAddress: compressed public key produces address starting with '1'
    - PubKeyToAddress: same key always produces same address (deterministic)
    - NewWallet: generates unique key pair and derives address
    - NewWallet: two wallets produce different addresses
    - Wallet stores and returns address, public key, private key
  </behavior>
  <action>
    1. Install dependencies: `go get github.com/btcsuite/btcd/btcec/v2 && go get golang.org/x/crypto/ripemd160`
    2. Create `internal/domain/wallet/base58.go`: Hand-rolled Base58 encode/decode using math/big division.
       - `Base58Encode(input []byte) string` -- big.Int division by 58, preserve leading zero bytes as '1' characters, reverse result
       - `Base58Decode(input string) []byte` -- reverse process, multiply by 58 and add
       - `Base58CheckEncode(version byte, payload []byte) string` -- version byte + payload + 4-byte double-SHA256 checksum
       - `Base58CheckDecode(input string) (version byte, payload []byte, err error)` -- decode, verify checksum, return version + payload
       - Use `base58Alphabet = "123456789ABCDEFGHJKLMNPQRSTUVWXYZabcdefghijkmnopqrstuvwxyz"` (no 0, O, I, l)
       - Reference code examples in RESEARCH.md but implement from scratch
    3. Create `internal/domain/wallet/address.go`: `PubKeyToAddress(pubKey *btcec.PublicKey) string`
       - Compressed public key (33 bytes) -> SHA-256 -> RIPEMD-160 (20 bytes) -> Base58CheckEncode with version 0x00
       - Also export `PubKeyHashFromAddress(address string) ([]byte, error)` for future use in tx validation (Base58CheckDecode, verify version 0x00, return 20-byte hash)
    4. Create `internal/domain/wallet/wallet.go`: Wallet entity following DDD patterns (unexported fields + getters).
       - `type Wallet struct` with address (string), privateKey (*btcec.PrivateKey), publicKey (*btcec.PublicKey)
       - `NewWallet() (*Wallet, error)` -- generate key pair via btcec.NewPrivateKey(), derive address
       - `ReconstructWallet(privKeyHex string) (*Wallet, error)` -- deserialize from hex, reconstruct pubkey and address
       - Getters: Address(), PrivateKey(), PublicKey(), PrivateKeyHex()
    5. Create `internal/domain/wallet/repository.go`: Repository interface
       - `Save(wallet *Wallet) error`
       - `GetByAddress(address string) (*Wallet, error)`
       - `ListAddresses() ([]string, error)`
    6. Create `internal/domain/wallet/errors.go`: `ErrWalletNotFound`, `ErrInvalidAddress`, `ErrInvalidChecksum`
