  <behavior>
    - TxInput: stores txID (block.Hash), vout (uint32), signature, pubKey; getters work
    - TxOutput: stores value (int64 satoshis), address (string); getters work
    - NewTransaction: creates transaction with given inputs and outputs, computes ID
    - Transaction.ComputeID: deterministic -- same inputs/outputs always produce same ID
    - Transaction.ComputeID: does NOT include signature in hash (chicken-and-egg avoidance)
    - Transaction.IsCoinbase: returns true for coinbase tx (zero hash input, 0xFFFFFFFF vout)
    - NewCoinbaseTx: creates coinbase with no real inputs, single output to miner address for given reward
    - NewCoinbaseTx: coinbase IsCoinbase() returns true
    - SignTransaction: signs all inputs with given private key, signature bytes set on each input
    - VerifyTransaction: valid signatures return true
    - VerifyTransaction: tampered transaction (changed output) returns false
    - VerifyTransaction: wrong key returns false
    - ValidateTransaction: rejects negative output values
    - ValidateTransaction: rejects if sum(outputs) > sum(inputs) when inputValues provided
    - ValidateTransaction: accepts if sum(outputs) == sum(inputs) (exact spend)
    - ValidateTransaction: accepts if sum(outputs) < sum(inputs) (implicit fee, difference is fee)
    - ValidateCoinbase: rejects coinbase with more than one output if reward doesn't match
    - ValidateCoinbase: accepts valid coinbase with correct reward
    - Change output: when input sum exceeds payment, a change output is auto-created to sender address
  </behavior>
  <action>
    1. Create `internal/domain/tx/input.go`: TxInput value object
       - Unexported fields: txID (block.Hash), vout (uint32), signature ([]byte), pubKey ([]byte)
       - `NewTxInput(txID block.Hash, vout uint32) TxInput` -- creates unsigned input
       - Getters: TxID(), Vout(), Signature(), PubKey()
       - Setters: SetSignature(sig []byte), SetPubKey(pk []byte) -- used during signing

    2. Create `internal/domain/tx/output.go`: TxOutput value object
       - Unexported fields: value (int64, satoshis), address (string, Base58Check)
       - `NewTxOutput(value int64, address string) TxOutput`
       - Getters: Value(), Address()

    3. Create `internal/domain/tx/transaction.go`: Transaction entity
       - Unexported fields: id (block.Hash), inputs ([]TxInput), outputs ([]TxOutput)
       - `NewTransaction(inputs []TxInput, outputs []TxOutput) *Transaction` -- creates tx, computes ID
       - `ReconstructTransaction(id block.Hash, inputs []TxInput, outputs []TxOutput) *Transaction` -- for loading from storage
       - `ComputeID() block.Hash` -- JSON serialize hashable struct (inputs without sigs, outputs) -> DoubleSHA256
         Use hashableTransaction/hashableInput/hashableOutput structs per RESEARCH.md Pattern 1.
         Signature and PubKey are EXCLUDED from hash computation.
       - `IsCoinbase() bool` -- single input with zero hash and 0xFFFFFFFF vout
       - Getters: ID(), Inputs(), Outputs()

    4. Create `internal/domain/tx/coinbase.go`: Coinbase factory
       - `NewCoinbaseTx(minerAddress string, reward int64) *Transaction`
       - Single input: zero Hash, vout=0xFFFFFFFF, no signature, no pubkey
       - Single output: reward amount to minerAddress
       - Compute and set ID

    5. Create `internal/domain/tx/signing.go`: ECDSA signing using btcec/v2/ecdsa
       - `SignTransaction(tx *Transaction, privKey *btcec.PrivateKey) error`
         For each non-coinbase input: compute tx ID (clean hash without sigs), sign with ecdsa.Sign(privKey, txHash.Bytes()), store sig.Serialize() and privKey.PubKey().SerializeCompressed() on input
       - `VerifyTransaction(tx *Transaction) bool`
         For each non-coinbase input: parse pubkey via btcec.ParsePubKey, parse sig via ecdsa.ParseSignature, verify sig.Verify(txHash, pubkey)
       - Skip verification for coinbase transactions (IsCoinbase check)

    6. Create `internal/domain/tx/validator.go`: Structural validation
       - `ValidateTransaction(tx *Transaction, inputValues []int64) error`
         - Reject if any output value <= 0
         - Reject if sum(outputs) > sum(inputValues) when inputValues provided
         - Reject if no inputs or no outputs (except coinbase)
       - `ValidateCoinbase(tx *Transaction, expectedReward int64) error`
         - Must have exactly 1 input with IsCoinbase() true
         - Must have exactly 1 output with value == expectedReward
       - `CreateTransactionWithChange(inputs []TxInput, inputValues []int64, toAddress string, amount int64, changeAddress string) (*Transaction, error)`
         - Sum inputValues, if sum < amount return ErrInsufficientFunds
         - Create output to toAddress for amount
         - If sum > amount, create change output to changeAddress for (sum - amount)
         - Return new transaction with computed outputs

    7. Create `internal/domain/tx/errors.go`: ErrInsufficientFunds, ErrInvalidSignature, ErrNegativeValue, ErrSumMismatch, ErrInvalidCoinbase

    Per user decision: int64 satoshis (1 coin = 100,000,000 satoshis), JSON serialization for hashing consistent with Phase 1.
