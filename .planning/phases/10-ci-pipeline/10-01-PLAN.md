<task type="auto">
  <name>Task 2: Create Go CI GitHub Actions workflow</name>
  <files>.github/workflows/ci-go.yml</files>
  <action>
Create `.github/workflows/ci-go.yml` with two parallel jobs: `test` and `lint`.

Workflow triggers: push to `master` branch AND pull_request (all branches).
NOTE: Default branch is `master`, NOT `main`.

**test job:**
```yaml
test:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v5
    - uses: actions/setup-go@v6
      with:
        go-version-file: go.mod
    - name: Run tests with coverage
      run: go test -coverprofile=coverage.out ./...
    - name: Display coverage
      run: go tool cover -func=coverage.out
```

**lint job:**
```yaml
lint:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v5
    - uses: actions/setup-go@v6
      with:
        go-version-file: go.mod
    - uses: golangci/golangci-lint-action@v9
      with:
        version: v2.10
```

Key points:
- `go-version-file: go.mod` reads Go version from the module file (no hardcoded version)
- `actions/setup-go@v6` handles module caching automatically
- Coverage is reported via `go tool cover -func=coverage.out` which prints per-function and total coverage (CI-05)
- `golangci-lint-action@v9` automatically finds `.golangci.yml` in repo root
- Test and lint jobs run in parallel (independent)

Create the `.github/workflows/` directory if it does not exist.
  </action>
  <verify>
    <automated>cat .github/workflows/ci-go.yml | head -5 && go test -coverprofile=coverage.out ./... && go tool cover -func=coverage.out | tail -1</automated>
  </verify>
  <done>`.github/workflows/ci-go.yml` exists with test (coverage) and lint jobs. Local `go test` with coverage and `golangci-lint run` both pass.</done>
