Create `.github/workflows/ci-frontend.yml` with a single `check` job.

Workflow triggers: push to `master` branch AND pull_request (all branches).
NOTE: Default branch is `master`, NOT `main`.

```yaml
name: Frontend CI

on:
  push:
    branches: [master]
  pull_request:

jobs:
  check:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: web
    steps:
      - uses: actions/checkout@v5
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
          cache-dependency-path: web/package-lock.json
      - run: npm ci
      - run: npm run lint
      - run: npx tsc -b
      - run: npm run build
```

Key points:
- `working-directory: web` set at job level via `defaults.run` so all `run` steps execute in `web/`
- `cache: npm` with `cache-dependency-path: web/package-lock.json` caches npm modules
- `npm ci` (not `npm install`) for clean, reproducible installs from lockfile
- `npx tsc -b` for TypeScript type checking (build mode, no emit)
- `npm run build` verifies the production build succeeds
- Node 22 matches the node:22-alpine used in web/Dockerfile

Create the `.github/workflows/` directory if it does not exist (it may already exist from plan 10-01).

Create `.github/workflows/docker.yml` with two parallel jobs: `build-backend` and `build-frontend`.

Workflow triggers: push to `master` branch AND pull_request (all branches).
NOTE: Default branch is `master`, NOT `main`.

CRITICAL: Add `permissions: contents: read, packages: write` at workflow level (GHCR requires `packages: write`).

**build-backend job:**
```yaml
build-backend:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v5
    - uses: docker/setup-buildx-action@v3
    - uses: docker/login-action@v3
      if: github.event_name == 'push'
      with:
        registry: ghcr.io
        username: ${{ github.actor }}
        password: ${{ secrets.GITHUB_TOKEN }}
    - uses: docker/metadata-action@v5
      id: meta
      with:
        images: ghcr.io/${{ github.repository }}
        tags: |
          type=ref,event=branch
          type=sha
    - uses: docker/build-push-action@v6
      with:
        context: .
        push: ${{ github.event_name == 'push' }}
        tags: ${{ steps.meta.outputs.tags }}
        labels: ${{ steps.meta.outputs.labels }}
        cache-from: type=gha
        cache-to: type=gha,mode=max
```

**build-frontend job** (same structure but different context/image name):
- `context: ./web` (frontend Dockerfile is in web/ subdirectory)
- `images: ghcr.io/${{ github.repository }}-web` (appends `-web` suffix)

Key points:
- `docker/login-action` only runs on push (not PR) -- PRs don't need registry auth
- `push` is conditional: `${{ github.event_name == 'push' }}` -- builds on PR but only pushes on master merge
- `docker/metadata-action@v5` generates tags: branch name (master) and short SHA (sha-abc1234)
- Buildx GHA cache (`cache-from/cache-to: type=gha`) for fast rebuilds
- Backend context is repo root (`.`), frontend context is `./web`
- Backend image: `ghcr.io/baotoq/shitcoin`, frontend image: `ghcr.io/baotoq/shitcoin-web`
